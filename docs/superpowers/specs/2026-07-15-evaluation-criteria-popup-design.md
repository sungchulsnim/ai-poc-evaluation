# 평가 기준 개편 및 설명 팝업 설계

## 목적

투표 기준을 기존 4개에서 다음 3개로 변경하고, 항목명을 누르면 상세 설명을 확인할 수 있게 한다.

| 항목 | 가중치 | 설명 |
|---|---:|---|
| 즉시활용성 | 50% | 법인이 바로 사용할 수 있는 완성도, 실사용 가능성, 입력/출력 흐름의 명확성 |
| 업무효과성 | 30% | 시간 절감, 반복업무 감소, 업무 품질 개선, 의사결정 지원 효과 |
| 사용편의성 | 20% | 법인에서 쉽게 이해하고 사용할 수 있는지, AI 결과 확인·수정이 쉬운지 등 |

## 투표 화면

각 과제의 평가 항목명과 가중치를 클릭 가능한 버튼으로 표시한다. 클릭하면 하나의 공용 HTML `dialog`에 항목명, 가중치, 설명을 표시한다. 닫기 버튼, ESC 키, 팝업 바깥 영역 클릭으로 닫을 수 있게 한다.

점수 선택 방식은 기존 5점~1점 라디오 버튼을 유지한다. 팝업 버튼은 점수 선택과 제출 이벤트에 영향을 주지 않는다.

## 데이터와 점수 계산

평가 항목 ID는 다음과 같이 사용한다.

- `immediateUse`: 즉시활용성 50%
- `businessEffect`: 업무효과성 30%
- `usability`: 사용편의성 20%

가중치 합계는 100%이며 최종점수는 기존과 동일하게 100점 만점으로 환산한다. 투표 설정 API의 원점수 범위는 가중 평균 기준인 4~20점, 최종점수 범위는 20~100점으로 일치시킨다.

서버 제출 검증, 결과 집계, 관리자 결과표, CSV 다운로드, 로컬 설정을 모두 같은 3개 기준으로 맞춘다.

## 기존 데이터

기존 4개 기준으로 저장된 제출 데이터는 새 기준과 호환되지 않으므로 새 버전 배포 직후 다음 항목을 초기화한다.

- `submissions`
- `submission_devices`

기존 관리자 초기화 기능 또는 동일한 D1 삭제 명령을 사용하며, 새 기준 배포가 확인된 뒤 실행한다.

## 변경 범위

- `config.json`
- `functions/api/[[path]].js`
- `public/admin.js`
- `public/vote.js`
- `public/app.css`
- 관련 자동 검증 스크립트

안내 화면, 관리자 로그인, 그룹별 제외 과제 설정, D1 바인딩 이름 `DB`는 변경하지 않는다.
